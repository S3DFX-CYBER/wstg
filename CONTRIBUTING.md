# WSTG में योगदान कैसे करें

WSTG (Web Security Testing Guide) में योगदान करने के लिए धन्यवाद!

आप इस प्रोजेक्ट में कई तरीकों से मदद कर सकते हैं। योगदान करने के लिए आपके पास एक GitHub अकाउंट होना चाहिए।

---

## आप कैसे योगदान कर सकते हैं

- नया कंटेंट लिखें (Author बनें)
- कंटेंट की समीक्षा करें (Reviewer / Editor बनें)
- नया Issue बनाएं
- Pull Request (PR) भेजें
- अनुवाद में योगदान दें

---

## Author (लेखक) कैसे बनें

अगर आप नया कंटेंट जोड़ना चाहते हैं या पुराने कंटेंट को अपडेट करना चाहते हैं:

1. पहले एक नया Issue बनाएं या कोई खाली Issue चुनें
2. उस Issue को अपने नाम पर असाइन करवाएं
3. नई ब्रांच बनाएं  
   उदाहरण:  
   `git checkout -b new-<issue-number>`
4. बदलाव करें और PR भेजें

---

## Reviewer / Editor कैसे बनें

अगर आप कंटेंट सुधारना चाहते हैं:

1. कोई open Issue चुनें (help wanted)
2. उस Issue पर comment करें और assign होने का अनुरोध करें
3. नई ब्रांच बनाएं  
   उदाहरण:  
   `git checkout -b fix-<issue-number>`
4. सुधार करें और PR भेजें

---

### Technical Review (तकनीकी समीक्षा)

अगर आपके पास सुरक्षा का अनुभव है, तो देखें:

- जानकारी सही और अपडेटेड हो
- Vulnerability और testing सही तरीके से समझाई गई हो
- जरूरी links शामिल हों
- कंटेंट तकनीकी लोगों के लिए उपयोगी हो

---

### Editorial Review (भाषा सुधार)

आप मदद कर सकते हैं:

- Grammar सुधारना
- Formatting ठीक करना
- शब्दों को सरल बनाना
- कंटेंट को छोटा और स्पष्ट करना

---

## Translation (अनुवाद)

WSTG सीधे अनुवाद को मुख्य repo में नहीं जोड़ता।

अगर आप अनुवाद करना चाहते हैं:

1. अपनी भाषा के लिए अलग GitHub repo बनाएं
2. अनुवाद पूरा करें
3. PDF बनाएं
4. WSTG repo में Issue खोलकर बताएं
5. वे आपके अनुवाद repo को सूची में जोड़ सकते हैं

---

## Issue कैसे बनाएं

1. GitHub पर नया Issue बनाएं
2. छोटा और स्पष्ट शीर्षक लिखें
3. बताएं:
   - क्या समस्या है
   - क्या सुधार चाहिए

उदाहरण:
- Typo या grammar गलती
- पुरानी जानकारी
- नया कंटेंट जोड़ने का सुझाव

---

## Pull Request (PR) कैसे भेजें

1. Repo को fork करें
2. Issue से अपने काम को जोड़ें
3. बदलाव करें
4. Style guide का पालन करें
5. अपने fork में changes push करें
6. PR बनाएं (Draft या Ready)

अगर काम पूरा नहीं है → Draft PR  
अगर review के लिए तैयार है → Normal PR

PR भेजने के बाद maintainers review करेंगे और जरूरत हो तो बदलाव मांगेंगे।

---

## PR merge होने की संभावना बढ़ाने के लिए

- Issue से लिंक करें
- Markdown lint errors ठीक करें
- Style guide का पालन करें
- Code सही और tested हो

---

## Contributor Environment Setup

1. GitHub अकाउंट बनाएं
2. Visual Studio Code इंस्टॉल करें
3. Markdown lint extension इंस्टॉल करें
4. Repo fork और clone करें

---

## GitHub Codespaces (वैकल्पिक)

आप cloud-based editor (Codespaces) का उपयोग करके भी योगदान कर सकते हैं।  
इससे setup आसान हो जाता है और सभी contributors का environment एक जैसा रहता है।
